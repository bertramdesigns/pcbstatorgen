## Current Codebase State

### What's DONE (from prior phases)
- **Python core decoupled & JSON-serializable** — `pcbstatorgen/serialization.py` provides pure functions: `config_to_dict`, `config_from_dict`, `force_result_to_dict`, `height_stack_to_dict`, `power_budget_to_dict`, `coil_paths_to_dict`. No GUI imports. ✓
- **Config hierarchy** — `BaseMotorConfig` → `LinearMotorConfig` (fully implemented) + `AxialMotorConfig` (stub, instantiable but geometry not implemented). `spacing_ratio` field EXISTS (config.py:284), drives `slot_pitch_m` (config.py:404), wired into ALL coil generators. ✓
- **Physics core** — `ForceEvaluator.evaluate()` returns `ForceResult` with `ripple_pct` property (force_eval.py:132). `MagnetArray`, `CoilCurrentModel`, `HeightStackCalculator`, `PowerEstimator`, `FrictionEstimator` all implemented and tested. ✓
- **Geometry generators** — 5 topologies: SERPENTINE, SINE_WAVE, CONCENTRATED, RHOMBIC, SPIRAL. All produce `PhaseCoil` objects. ✓
- **JSON-RPC server** — `pcbstatorgen/server.py` exposes: `ping`, `build_config`, `estimate_force`, `evaluate_force`, `sample_bfield`, `coil_geometry`, `height_stack`, `power_estimate`. ✓
- **Test suite** — 490 tests covering config, geometry, magnetic physics, units, via grid, KiCad connection. Offline-capable (`pytest -m "not slow and not integration"`). ✓

### What's BROKEN / MISSING
- **Streamlit app is non-functional** — `gui/streamlit_app.py` (890 lines) has 13+ undefined symbols (`build_config`, `render_output_tabs`, `_nav_buttons`, `_step1_envelope`, `_step3_magnet_arrangement`, `_step5_pcb_tier`, `_FADER_PRESETS`, etc.). The script raises `NameError` on run.
- **Wizard is partially scaffolded** — 6-step wizard skeleton exists (lines 392–422) but only 3 of 6 step renderers are defined. PRODUCT_GOALS says wizard should be REMOVED entirely.
- **Magnet grade helper not in core** — `magnet_remanence_t` is a raw scalar. No N35–N52 lookup table exists in `pcbstatorgen/`. The UI has `_MAGNET_BR_LOOKUP` (lines 58–66) but it's NOT wired to `magnet_remanence_t` in config construction.
- **Ripple not wired to UI** — `ForceResult.ripple_pct` exists in core but `st.session_state["force_result"]` is never set. The "Run Force/Torque Preview" button doesn't exist.
- **No UI tests** — 0 tests for `gui/streamlit_app.py`.
- **No Vernier ratio tests** — `spacing_ratio` field exists and is wired, but no test exercises 4:5 or 5:6 ratios.
- **`render_output_tabs()` undefined** — Referenced at line 891 but never implemented. Needs coil geometry preview, force plot, B-field plot, export.
- **Friction/normal force not displayed** — `FrictionEstimator` exists but friction is not shown as a metric. No stacked bar chart for Net Usable Force Margin.
- **Magnet gap vs. pitch reconciliation** — Core has `magnet_pitch_m` as direct input, `magnet_gap_m` as derived. PRODUCT_GOALS spec wants gap as primary UI input with `pole_pitch = magnet_width + gap` computed dynamically.

## Work Item Breakdown

### WI-1: Magnet Grade Lookup Table (CORE)
**Priority**: HIGH | **Subagent**: `general` | **Depends on**: None

Add a magnet grade reference module to the Python core:
- Create `pcbstatorgen/magnet_grades.py` with a lookup table for N35–N52 grades
- Each entry: grade name, Br min/typ/max (T), max operating temp by suffix (Std/H/SH/UH/EH/AH)
- Provide `get_remanence(grade: str) -> float` returning typical Br
- Provide `MAGNET_GRADES` dict/list for UI consumption
- Add `magnet_grade` optional field to `BaseMotorConfig` (stored alongside `magnet_remanence_t`)
- Add tests in `tests/test_magnet_grades.py`
- **Files**: `pcbstatorgen/magnet_grades.py` (new), `pcbstatorgen/config.py` (add field), `tests/test_magnet_grades.py` (new)

### WI-2: Vernier Spacing Ratio Tests (TESTS)
**Priority**: HIGH | **Subagent**: `general` | **Depends on**: None

Add test coverage for the existing `spacing_ratio` feature:
- Test `slot_pitch_m` with ratios 1.0, 0.8 (4:5), 0.833 (5:6)
- Test `WaveWindingGenerator.conductor_x_positions` with non-1.0 ratios
- Test that Vernier ratios produce different conductor layouts than 1:1
- Test ripple reduction: run `ForceEvaluator.evaluate()` with 1:1 vs 4:5, assert ripple_pct changes
- **Files**: `tests/test_vernier_ratios.py` (new)

### WI-3: Streamlit Wizard Removal & Dashboard Consolidation (UI)
**Priority**: HIGH | **Subagent**: `streamlit-expert` | **Depends on**: WI-1 (for magnet grade wiring)

Complete overhaul of `gui/streamlit_app.py`:
- **Remove all wizard code**: `render_wizard()`, `_step0_application`, `_step2_coil_topology`, `_step4_force_and_feel`, `_nav_buttons`, `_render_review_card`, `_STEP_TITLES`, `_N_STEPS`, `build_config_from_wizard`
- **Fix undefined references**: Implement `build_config()` as the sole config builder (rebased on `build_config_from_advanced`), implement `render_output_tabs(config)`
- **Single dashboard layout**: Topology selector toggle (Linear/Radial) at top, parameter panel (left), live feedback panel (right), output tabs at bottom
- **Parameter renames**: "Travel" → "Center-to-Center Travel (mm)", "Board width" → "Active Area Width (mm)"
- **Magnet gap as primary input**: Replace "Magnet Pitch" slider with "Gap between magnets" slider; compute `pole_pitch = magnet_width + gap` and display as derived metric
- **Wire magnet grade dropdown**: Selecting a grade updates `magnet_remanence_t` in config
- **Wire ripple computation**: Add "Run Force/Torque Preview" button that calls `ForceEvaluator.evaluate()`, stores `ForceResult` in `st.session_state["force_result"]`, displays `ripple_pct`
- **Friction/force stacked bar**: Display electromagnetic force, friction, and Net Usable Force Margin as stacked bar chart
- **Zero-cogging block**: Keep existing explanation, ensure it's always visible
- **Files**: `gui/streamlit_app.py` (major rewrite)

### WI-4: Output Tabs Implementation (UI)
**Priority**: MEDIUM | **Subagent**: `streamlit-expert` | **Depends on**: WI-3

Implement `render_output_tabs(config)` with tabs for:
- **Coil Geometry Preview**: SVG/matplotlib rendering of coil paths via `coil_paths_to_dict()`
- **Force vs. Position**: Plot `ForceResult.force_x_n` vs `positions_m` from physics sweep
- **B-Field Profile**: Plot `MagnetArray.bfield_at_pcb_surface()` along travel axis
- **Height Stack**: Keep existing matplotlib barh chart
- **KiCad Export**: Placeholder button (Phase 5+ implements actual writer)
- **Files**: `gui/streamlit_app.py` (add function)

### WI-5: Active Zone Length Feedback (UI)
**Priority**: MEDIUM | **Subagent**: `streamlit-expert` | **Depends on**: WI-3

Add real-time feedback for the Center-to-Center Travel → Active Zone Length relationship:
- Display `L_active = L_travel + coil_span` as a derived metric
- Visual indicator showing mover coverage across the active zone
- **Files**: `gui/streamlit_app.py` (add to feedback panel)

## Risks & Mitigations

| Risk | Impact | Mitigation |
|---|---|---|
| Streamlit app rewrite is large (890→~600 lines) | High | Break into WI-3 (core consolidation) + WI-4/5 (features). Test incrementally. |
| `AxialMotorConfig` is a stub | Medium | Phase 4 UI should support the toggle but gracefully handle unimplemented radial geometry. Don't attempt to implement radial geometry generation in Phase 4. |
| Vernier ripple reduction claims unvalidated | Medium | WI-2 tests will validate whether 4:5/5:6 ratios actually reduce ripple. If not, adjust product claims. |
| Magnet gap/pole_pitch refactor could break downstream | Low | Keep core as-is (`magnet_pitch_m` primary). Do the gap→pitch computation at the UI layer only. |
| No UI test framework | Low | Focus on core tests (WI-2). UI testing via manual verification. Streamlit testing is out of scope for Phase 4. |
